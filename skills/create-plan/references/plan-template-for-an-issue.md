# Plan Template For An Issue

各 Issue に対して以下テンプレートで「計画作成用プロンプト」を生成する。
`{...}` は実値に置換する。該当情報がない行は削除する。
`{plan_name}` は親 plan ファイル名（`docs/plans/{name}.md` の `{name}`）を使う。

## 共通テンプレート

````
```
{実行モード宣言行}
{ベースブランチ} から {planブランチ名} を作成して、Issue #{Issue番号} の計画書を作ってください。

まず `gh issue view {Issue番号}` で Issue を読んでください。
{補足ドキュメント行（省略可）}
{既存コード・既存文章の参照行（省略可）}
{技術制約・執筆制約行（省略可）}

成果物として `docs/plans/{plan_name}/issues/{Issue番号}-{slug}.md` を作成してください。
計画には以下を必ず含めてください。
1. 目的とスコープ（何をやる/やらない）
2. 現状整理（既存コード・既存文章・前提）
3. 実施ステップ（順序つき）
4. 変更対象（ファイルパス、または章・節の単位）
5. トレードオフと採用理由
6. リスクと回避策
7. 完了条件（DoD）
8. チェックボックス付き ToDo

不明点や不足情報は、作業開始前に質問として列挙してください。
まだ実装・執筆本文の確定版作成はしないでください。計画書の作成だけ行ってください。

将来の実行方法も末尾に1段落で記載してください。
- コード実装なら `/impl-single` または `/impl-multi` での実行想定
- 論文作業なら章ごとの執筆・推敲の進め方
```
````

## 実行モード宣言行

Issue の性質に応じて先頭行を切り替える。

- Java実装系:
  - `Java実装計画モードで進めてください。`
- 科学論文系:
  - `科学論文計画モードで進めてください。`
- 混在/不明:
  - `技術計画モードで進めてください。`

## 補足ドキュメント行の生成ルール

Issue本文に `docs/`, `research`, `plan`, `SPEC-` 参照がある場合のみ生成。

例:

- `補足として docs/SPEC-rough-estimate.md を参照してください。`
- `補足として research_20260319_topic.md の「Related Work」節を参照してください。`
- `補足として docs/SPEC-final-estimate.md と plan_20260319_xxx.md を参照してください。`

## 既存コード・既存文章の参照行

Issue本文に参照先の明示がある場合のみ生成。

例（Java）:

- `既存の src/main/java/com/example/analysis/ 配下を読み、同じ設計パターンを前提に計画してください。`
- `既存の build.gradle と test クラス構成を確認して計画に反映してください。`

例（論文）:

- `既存の docs/paper/draft.md の構成を引き継ぎ、差分方針を計画に含めてください。`
- `既存の図表一覧（docs/paper/figures.md）を確認して、追加・修正方針を計画してください。`

## 技術制約・執筆制約行

Issue本文に明示された制約がある場合のみ生成。

例:

- `Java 21 と Spring Boot 3 の範囲で計画してください。`
- `統計解析は再現可能性を重視し、入力データと検証手順を計画に明記してください。`
- `論文は IMRaD 構成（Introduction/Methods/Results/Discussion）を維持してください。`
